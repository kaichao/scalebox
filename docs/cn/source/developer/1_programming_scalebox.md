## 1.1 Scalebox程序

- 面向数据密集型场景优化的容器化计算程序
- 依赖于Scalebox执行框架，通过消息驱动形式实现多节点分布式、并行计算
- 并行方式
  - 模块级数据并行
  - 模块间流水线并行
  - 模块内代码并行
- 主要应用于天文计算场景，未来可应用于生物数据处理、数字电影制作、视频加工等高数据I/O场景。

表：Scalebox与传统MPI编程的比较

| 特性	     | Scalebox	     | 传统的MPI/Charm++                  |
| -------   | ------------- | -------------------------|
| 编程范式   | 消息驱动、过分解 | 进程通信、单程序多数据(SPMD) |
| 负载均衡	 |动态、自动化     | 多为主观、静态              |
| 容错机制	| 内置支持        | 要额外实现                |

| 编程复杂度 |	高级抽象，较低  |	底层控制，较高            |

## 1.2 Scalebox应用程序的组成

Scalebox应用程序由多个容器化封装的软件模块组成。其中包括：
- 主路由模块：系统主程序，负责将任务分配给定制算法模块、公共基础模块，并维护整个应用的运行状态。
- 定制算法模块：用户业务模块，以消息驱动形式，实现特定算法功能。
- 公共基础模块：
  - 文件传输：基于计算系统内置的ssh，支持跨节点的数据传输
  - 目录列表：辅助产生待处理文件


- Scalebox应用程序的应用定义文件：
  - 应用模版文件：以yaml文件格式，定义应用所包含的各个模块的相关定义
  - 参数定义文件：以环境变量文件形式，定义应用模块文件中相关参数



## 1.3 核心概念与编程模式

| 概念     | 英文名称   | 中文名称 | 说明                                                     |
| :------ | :--------- | :------ | :------------------------------------------------------ |
| 应用程序 | **App**    | **应用** | 完成特定计算任务的应用程序，包含多个Module。App通常通过调用资源调度系统（如Slurm）动态获取所需的计算资源。 |
| 软件单元 | **Module** | **模块** | 构成应用程序App的基本可编程单元，是以容器化封装的程序代码集合。通常为一个独立的算法组件，或者是跨节点之间的传输组件等。组件间通过级联形成数据处理流水线。|
| 执行实例 | **Task**   | **任务** | 某个Module根据输入数据产生的具体执行实例。是运行的基本单位。 |
| 计算节点 | **Host**   | **节点** | 执行计算任务的服务器节点。                                 |
| 资源单元 | **Slot**   | **插槽** | 节点上与Module对应的计算资源切片，是Task细粒度运行调度的基础单位  |

# 1. 编程模型

表：Scalebox与传统MPI编程的比较

| 特性	     | Scalebox	     | 传统MPI                  |
| -------   | ------------- | -------------------------|
| 编程范式   | 消息驱动、过分解 | 进程通信、单程序多数据(SPMD) |
| 负载均衡	 |动态、自动化     | 多为主观、静态              |

| 可扩展性	| 流式驱动，<br/>支持极大规模并行 | 大规模时面临挑战  |
| 编程复杂度 |	高级抽象，较低  |	底层控制，较高            |

## 1.1 核心概念与编程模式

| 概念     | 英文名称   | 中文名称 | 说明                                                     |
| :------ | :--------- | :------ | :------------------------------------------------------ |
| 完整应用 | **App**    | **应用** | 提交给资源调度系统（如Slurm）的完整计算任务。由多个Block组成。   |
| 软件单元 | **Module** | **模块** | 构成应用的基本可编程单元，一个独立的算法组件。通过级联形成数据处理流水线。|
| 执行实例 | **Task**   | **任务** | 某个Module根据输入数据产生的具体执行实例。是运行的基本单位。 |
| 计算节点 | **Host**   | **节点** | 执行计算任务的服务器节点。                                 |
| 资源单元 | **Slot**   | **插槽** | 节点上与Mod对应的计算资源切片，是Task细粒度运行调度的基础单位  |

## 标准模块的可编程特性

- 标准模块：其功能脚本可放在/app/share/bin下，子模块的功能脚本在/app/bin下。模块识别规范是优先使用/app/bin，再搜索/app/share/bin目录下；
- message-router消息格式定制：按标准模块的消息格式定制，便于使用标准模块功能；

这样可充分利用标准模块的功能。
